# Stage 6 — create_boss simplification (profile_id + seed)

## Goal

Simplify the `dev/create_boss` flow so that a single boss is generated by **profile_id** and **seed**. Output is standardized to profile_id, name, description, region, level, and stats. Stage 5’s DoD is superseded by this stage.

## Non-goals

- No quest level, Rank, Overall, or weighted profile selection.
- No change to data file schemas beyond what is already in use (`data/boss_profiles.json`, `data/levels.json`).

## Definition of Done (DoD)

- **Core** exposes `GenerateBoss(profileID string, seedOpt *int64) (Boss, error)` that:
  - Looks up the profile by id across all regions (e.g. via a function like `ProfileByID`).
  - Uses the seed to determine **level** (uniform draw from the profile’s `level_min`..`level_max`) and **variation** (ratio perturbation).
  - Returns a boss value with: ProfileID, Name, Description, Region, Level, Stats (Power, Guard, Evasion, Cunning).
- **REPL**: `dev/create_boss <profile_id> [seed]`
  - `profile_id` is required; missing or empty is an error.
  - `seed` is optional; when omitted, a time-based seed is used.
  - Prints output as defined in `04_TerminalUI.md` (profile_id, name, description, region, level, stats).
- `go test ./...` passes.

## Required Spec Updates

Updates are applied to the owning specs so they match this stage; the stage doc does not duplicate them.

- **02_System.md** — “Boss monster generation” section: describe profile-by-id lookup, level from profile’s level_min/level_max driven by seed, stat budget from `monster_level_budget_model`, ratio + variation, and output fields (profile_id, name, description, region, level, stats only). Remove Quest Level, Rank, Overall, and weighted selection.
- **03_Balance.md** — “Boss generation”: remove QUEST_LEVEL_MONSTER_LEVEL_RANGE, OVERALL_RATING_MODEL, BOSS_PROFILE_RANK. Add BOSS_PROFILE_LEVEL_RANGE (per-profile level_min/level_max). Keep BOSS_PROFILES_FILE, BOSS_PROFILE_VARIATION, MONSTER_LEVEL_STAT_BUDGET_MODEL.
- **04_TerminalUI.md** — “Dev commands”: command `dev/create_boss <profile_id> [seed]`, output contract (profile_id, name, description, region, level, stats), and example output.

## Test Checklist

- **Core**
  - For a valid `profile_id` and fixed `seed`, `GenerateBoss` returns a boss with the expected profile and a level within that profile’s level_min..level_max; the same (profile_id, seed) produces identical output across runs.
  - Invalid or unknown `profile_id` returns an error (no panic).
- **REPL**
  - `dev/create_boss <profile_id>` with no seed prints a boss summary containing profile_id, name, description, region, level, and stats.
  - `dev/create_boss <profile_id> <seed>` with the same arguments reproduces the same output.
  - Missing profile_id or invalid profile_id yields an error message.
