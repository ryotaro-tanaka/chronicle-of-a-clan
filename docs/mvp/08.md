# Stage 08 — Data layout and combat spec split

## Goal
- Reorganize read-only data under `data/quests/`, `data/combat/`, and `data/items/`.
- Keep combat documentation within the four canonical specs (`00`–`04`): rules in `02_System.md`, numbers/formulas in `03_Balance.md`, data paths in `01_Dev.md`, high-level intent in `00_Overview.md`.

## Non-goals
- Implementing quest-day combat resolution in code.
- Adding a fifth top-level spec file (e.g. no separate combat-only doc).

## Definition of Done
- Loaders use the new paths; data files live under the three subtrees above.
- `02_System.md` states MVP boss-hunt rules (no Momentum, stat mapping, tick steps).
- `03_Balance.md` holds all daily combat constants, party-average definitions, budget curves, and reference-armor formula.
- `01_Dev.md` documents each JSON file’s role; `00_Overview.md` states doc boundaries and brief combat design intent without numbers.

## Required Spec Updates
- **`00_Overview.md`**, **`01_Dev.md`**, **`02_System.md`**, **`03_Balance.md`** per the split above.
