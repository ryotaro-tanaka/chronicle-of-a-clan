# Stage 5 — Boss Generation & dev/create_boss

## Goal

- Implement data-driven boss generation in Core using `data/boss_profiles.json` and `data/levels.json`, and expose it via a development-only REPL command `dev/create_boss` that generates and prints a single boss for inspection.

## Non-goals

- No quest list generation or quest browsing UI.
- No rank unlock flow (e.g., clearing Rank 1 to unlock Rank 2).
- No reward tables (gold/fame/materials) or drop logic.
- No member level/stat progression logic (the `member_level_budget_model` is out of scope for this stage).

## Definition of Done (DoD)

- Core exposes a function (e.g. `GenerateBoss(region string, questLevel int, seedOpt *int64)`) that:
  - Uses `data/boss_profiles.json` to select a boss profile and apply variation.
  - Uses `data/levels.json` to determine the monster level range for the given quest level and to compute a total stat budget for the chosen monster level.
  - Returns a boss object containing at least: `Region`, `QuestLevel`, `MonsterLevel`, `Rank`, `Overall`, `ProfileID`, `Name`, and `Stats` (Power/Guard/Evasion/Cunning).
- In the REPL, running `dev/create_boss [region] [quest_level] [seed]`:
  - With no args, defaults to `region=forest`, `quest_level=1`, and an internal random seed.
  - When provided, `region` and `quest_level` are parsed and passed through to Core; `seed` (if present) is used for reproducible generation.
  - Prints a single boss summary including Region, QuestLevel, MonsterLevel, Rank, ProfileID, Name, Overall, and Stats.
- `go test ./...` passes.

## Required Spec Updates

- **02_System.md**
  - Add a “Boss monster generation” section that defines:
    - Boss Profiles as 4-stat ratio patterns plus `rank` (fixed tier) and `name` (display name).
    - Region-scoped weighted selection of a profile from `data/boss_profiles.json` and application of per-region variation based on the number of focused stats.
    - The flow: QuestLevel → MonsterLevel range → MonsterLevel → total stat budget → distribution into Power/Guard/Evasion/Cunning using the selected profile and variation.
    - The Overall rating rule: given a MonsterLevel and its QuestLevel’s allowed range, compute a 1..`steps` value indicating under/normal/over strength within that range.
    - The distinction between **Rank** (fixed tier used for progression) and **Overall** (relative roll inside that tier/quest level).
- **03_Balance.md**
  - Document the following parameters and their backing files:
    - `BOSS_PROFILES_FILE` → `data/boss_profiles.json`.
    - `BOSS_PROFILE_VARIATION` → per-region `variation[]` in `boss_profiles.json`.
    - `BOSS_PROFILE_RANK` → per-profile `rank` in `boss_profiles.json`.
    - `QUEST_LEVEL_MONSTER_LEVEL_RANGE` → `quest_levels[]` in `data/levels.json`.
    - `MONSTER_LEVEL_STAT_BUDGET_MODEL` → `monster_level_budget_model` in `data/levels.json`.
    - `OVERALL_RATING_MODEL` → `overall_rating` in `data/levels.json`.
- **04_TerminalUI.md**
  - Add a dev-only command contract:
    - Command: `dev/create_boss [region] [quest_level] [seed]`
      - `region` (optional, default `forest`)
      - `quest_level` (optional, default `1`)
      - `seed` (optional, used for reproducible results when provided)
    - Output: a human-readable one- or two-line summary including Region, QuestLevel, MonsterLevel, Rank, Overall, ProfileID, Name, and Stats (Power/Guard/Evasion/Cunning).

## Test Checklist

- **Core**
  - For a valid `(region, questLevel)` pair, `GenerateBoss` returns a boss with:
    - A profile drawn from that region’s profiles.
    - A `MonsterLevel` within the configured range for that `questLevel`.
    - A total stat budget consistent with the `monster_level_budget_model`.
  - If seed support is used, the same `(region, questLevel, seed)` triple produces identical results across runs.
  - Invalid regions or quest levels cause an error (not a panic).
- **REPL**
  - `dev/create_boss forest 1` prints a boss summary containing Region/QuestLevel/MonsterLevel/Rank/ProfileID/Name/Stats.
  - If `seed` is supported and provided, repeating the same command reproduces the same summary.
