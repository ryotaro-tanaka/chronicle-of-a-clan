# Stage 07 — Quest list and detail

## Goal
Implement quest list (under `quests/keys` and `quests/<region>`) and quest detail view (`info`) that shows boss profile data. Listing is driven by save `key_quest_progress.current_order`, `data/key_quests.json`, and `data/boss_profiles.json`.

## Non-goals
- Accepting or completing quests (state changes).
- Rewards display (remain blank).
- Non-key quests or minor monsters.

## Definition of Done
- `ls` at root shows `[DIR] quests` (with clan, dev, exit).
- `ls quests/` shows `[DIR] keys`, `[DIR] forest`, `[DIR] volcano`, `[DIR] desert`, `[DIR] tundra`, `[DIR] swamp`.
- **`ls quests/keys/`**: Lists only the key quest whose `order` equals `current_order` (the next quest to advance the story). One entry in practice (one quest per order).
- **`ls quests/<region>/`**: Lists all key quests with `order <= current_order` in that region. Each entry: `[DIR] hunt_<monster_slug>`.
- Under each `hunt_<slug>`: `[VIEW] info`.
- Invoking `info` (e.g. `quests/keys/hunt_ambushjaw_gator/info`) prints: Name, Lv: `<min>-<max>`, Specialties: `<stats>`, Reward: (empty), Description: `<description>`.
