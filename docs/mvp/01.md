# Stage 1 — Interactive skeleton: Startup load + Status view

## Goal
Provide a minimal interactive program that can:
- start an interactive session
- load a save folder from disk at startup (via CLI argument)
- display clan status via a dedicated view command

This stage establishes the end-to-end “walking skeleton” for portability and basic UX.

## Non-goals
- No in-session load/switch of save folders
- No quest generation or quest listing
- No hiring/dismissal
- No equipment crafting/upgrade flows
- No time advancement
- No balancing work
- No `help` command

## Definition of Done (DoD)
- The program starts an interactive session when invoked with a save directory argument.
- The program loads the save folder at startup.
- The user can view clan status after startup load.
- The save folder can be copied to another machine and results in the same displayed state.
- Startup fails with clear errors for:
  - missing save directory argument
  - invalid save directory path (not found / not a directory)
  - missing required file(s) in the save folder
  - invalid JSON
  - unsupported `save_version`
- Minimal automated tests exist for successful startup load and expected failure modes.

## User-facing flow (Stage 1)
- `./myapp` (no args) prints an actionable message and exits.
- `./myapp <save_dir>` loads the save and enters the interactive prompt.

## Implementation Deliverables
- Startup argument parsing:
  - `./myapp <save_dir>` is the canonical invocation.
- Startup load:
  - parse required save file(s) from `<save_dir>`
  - error clearly on failure
- Interactive prompt with at least:
  - `ls`
  - `cd <path>`
  - `status` (view)
  - `exit` (action)
- Virtual FS navigation:
  - `ls` shows available entries with type tags: `[DIR]`, `[VIEW]`, `[ACT]`
  - relative path execution is supported (e.g., `../exit`)

## Test Checklist
- No-arg startup: exits with expected message.
- Invalid path: exits with expected message.
- Valid save folder loads successfully and enters prompt.
- Missing `clan.json` produces the expected error.
- Invalid JSON in `clan.json` produces the expected error.
- Unsupported `save_version` produces the expected error.
- `status` output includes all required fields (even if some sections are empty).
