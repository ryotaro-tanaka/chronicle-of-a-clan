# Stage 09 — Bubble Tea unified UI and party setup

## Goal
- Replace the remaining `go-prompt`-style REPL interaction with a Bubble Tea-based terminal UI.
- Add a pre-departure party setup flow for quests: select up to 4 members, then choose weapon and armor for each selected member.
- Keep the existing virtual FS navigation model (`ls`, `cd`, `status`, `exit`, path invocation) intact.

## Non-goals
- Persisting quest party selections to save files.
- Implementing quest departure, day progression, or combat execution.
- Reintroducing advanced shell-like features beyond MVP line editing and tab completion.

## Definition of Done
- `coc <save_dir>` launches a Bubble Tea UI; `go-prompt` is no longer required.
- Quest directories expose `[ACT] party` and `[ACT] clear` alongside `[VIEW] info`.
- `party` opens a member-selection screen, then an equipment-selection screen, and returns to navigation with the selection kept in session memory.
- `clear` removes the stored party selection for the current quest only.
- Member level, base stats, and equipment candidates are derived from `clan.json` plus the read-only combat/item JSON files.

## Required Spec Updates
- **`00_Overview.md`**: reflect the unified TUI and quest preparation loop at a high level.
- **`01_Dev.md`**: record Bubble Tea as the interactive UI framework and note session-scoped party state.
- **`04_TerminalUI.md`**: document the Bubble Tea navigation model, `party` / `clear`, and the party/equipment screen contracts.
