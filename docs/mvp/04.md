# Stage 4 — Fix REPL input: real go-prompt (tab completion + exit)

## Goal
Fix the REPL so that:
- Tab completion actually works at runtime
- `exit` reliably terminates the interactive prompt
by switching from the local minimal prompt implementation to the real `github.com/c-bata/go-prompt`.

## Non-goals
- No new gameplay features
- No new commands
- No TUI screens
- No help command

## Background (current bug)
- The repository currently replaces `github.com/c-bata/go-prompt` with a local minimal implementation under `third_party/go-prompt`.
- That local implementation reads lines via `bufio.Scanner` and does not handle key events (Tab), nor does it stop the loop after `exit`.

## Definition of Done (DoD)

### Replace the prompt implementation
- Stop using the local `third_party/go-prompt` replacement.
- Use the real `github.com/c-bata/go-prompt` so that key events and completers work.

### Tab completion works
- Pressing Tab triggers the existing completer logic and shows suggestions.
- Completion covers:
  - command names: `ls`, `cd`, `status`, `exit`
  - virtual FS paths (e.g., `cd clan/`, `ls clan/`, `clan/status`)

### Exit works
- `exit` ends the interactive session.
- Path-based invocation that resolves to the exit node (e.g., `../exit`, `clan/../exit`) also exits.
- The REPL must not continue showing `>` after exit.

### Tests
- `go test ./...` passes.
- Add/adjust a focused test to ensure:
  - the “exit condition” is reachable from the same resolution logic used by the REPL
  - completion candidates are produced for at least one representative input

## Notes
- Prefer using go-prompt’s built-in exit checking mechanism rather than re-implementing the prompt loop.
