# Stage 2 — Core package rename + file split + ls <path> + go-prompt completion + status polish

## Goal
Turn the Stage 1 prototype into a maintainable foundation and a more Unix-friendly UX by:
- adopting a standard-ish Go repo layout
- splitting the oversized `main.go` by responsibility
- keeping the virtual FS navigation model while adding:
  - `ls <path>` (list without changing location)
  - line editing + tab completion via `go-prompt`
- polishing `status` output (compact, readable)
- aligning usage/errors with the binary name `coc`

## Non-goals
- No quest generation or quest listing
- No hiring/dismissal
- No equipment crafting/upgrade flows
- No time advancement
- No balancing work
- No TUI screens (selection UI may come later)
- No `help` command

## Decisions (Stage 2 makes these official)
- Core logic package naming: use `internal/core/` (not `internal/game/`).
- Interactive input library: use `go-prompt` for history, line editing, and tab completion.
- Keep the virtual FS model as the authoritative navigation structure.

## Definition of Done (DoD)

### Repo layout (minimal target)
- Entry point: `cmd/coc/main.go`
- Core/domain + persistence: `internal/core/`
- Terminal UI + virtual FS: `internal/ui/` (or similar; exact names can vary)
- Tests move with the logic (no giant `main_test.go` for everything)

Recommended minimum tree:
- `cmd/coc/main.go`
- `internal/core/save/` (load/validate save, schema structs)
- `internal/ui/repl/` (prompt loop, go-prompt integration)
- `internal/ui/vfs/` (node tree, path resolution, listing)
- `docs/`
- `examples/`

### File splitting (main.go reduction)
- `cmd/coc/main.go` contains only:
  - arg parsing
  - calling into core save loader
  - starting the REPL
- Save loading/validation moves out of main.
- VFS tree + path resolver moves out of main.
- Status formatting moves out of main.

### UX improvements
- `ls` supports an optional path argument:
  - `ls` lists the current location.
  - `ls <path>` lists the target location without changing the current location.
- `status` output is compact:
  - `Clan: <name>   Day: <day>`
  - `Gold: <gold>   Fame: <fame>`
  - `Members: <count>   ActiveQuests: <count>`
- Usage/errors mention `coc` (or use basename of argv[0]) instead of hard-coded `./myapp`.

### go-prompt integration
- Interactive input uses `go-prompt`.
- Tab completion exists for:
  - command names: `ls`, `cd`, `status`, `exit`
  - virtual FS paths relevant to the current input:
    - after `cd `: directory paths
    - after `ls `: directory paths
    - path invocation like `clan/status`
- Completion candidates are derived from the virtual FS tree and current location (no separate source).

### Minimal tests
- VFS/path resolver tests:
  - `ls <path>` lists target without changing cwd
  - relative paths: `..`, `.`
  - simple normalization (accept `clan` and `clan/`)
- Status formatting test:
  - required lines exist and values match loaded state
- Usage/binary name test:
  - usage includes `coc` or argv[0] basename

## Notes / Open Questions
- Whether `status` should be visible at root or only under `clan/` (recommended: only under `clan/` to keep root minimal).
- Chronicle JSONL line length assumptions (if lines can be large, avoid Scanner defaults).
